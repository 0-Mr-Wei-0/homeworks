1、web URL
HTML页面
	1.主页
		1)未登录主页
		url: /
		作用：用于显示尚未登录状态的显示页面，提示选择登录、注册
		1)登录主页
		url: /main
		作用：用于显示登录状态的显示页面，提示选择发邮件、读邮件、登出
	2.用户
		1)登录
		url: /user/login
		作用：用于显示用户登录页面
		2)注册
		url: /user/register
		作用：用于显示用户注册页面
	3.邮件
		1)列表
		url: /mail/list
		作用：用于显示邮件列表页面
		2)发送
		url: /mail/send
		作用：用于显示邮件发送页面
		3)内容	
		url: /mail/content
		作用：用于显示邮件内容页面
	4.退出
	url: /out
	作用：退出登录，显示未登录页面
2、web API
JSON数据处理
	1.用户
		1)登录
		url: POST /users/login
		作用：用于接收用户登录信息的表单，并对信息进行处理，然后返回“登录成功/用户名或密码错误”到前台
		2)注册
		url: POST /users/register
		作用：用于接收用户注册信息的表单，并对信息进行处理，然后返回“注册成功/改用户已存在”到前台
	2.邮件
		1)列表
		url: GET /mails
		作用：返回邮件列表数据到前台
		2)发送
		url: POST /mails/send
		作用：用于接收邮件发送表单，并对信息进行处理，然后返回“发送成功/该用户不存在”到前台
		3)内容	
		url: GET /mails/:id
		作用：返回具体邮件（指定id）内容数据到前台
