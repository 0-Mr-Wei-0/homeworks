1.Web URL
	1.首页
		url:/home
		作用：提供给用户选择注册或是登录的主页面

	2.注册
		url:/user/regist
		作用：接收用户输入的注册信息

	3.注册成功
		url:/user/regist/success
		作用：用户注册成功，跳转到注册成功页面

	4.登录
		url:/user/login
		作用：接收用户输入的登录信息

	5.登录成功
		url:/user/login/success
		作用：用户登录成功，跳转到登录成功页面，用户可以选择读写邮件的功能

	5.写邮件
		url:/email/write
		作用：接收用户输入的邮件内容

	6.发送成功
		url:/email/send
		作用：跳转到发送成功页面

	7.查看邮件
		url:/email/read
		作用：列出邮件列表，供用户选择


2.Web API
	1.注册成功与否的判断
		url:/users
		method:post
		data:action=regist
		作用：根据该用户输入的信息，查询数据库中的user表，判断该用户输入的数据是否与数据库中的信息冲突，返回查询结果

	2.登录成功与否的判断
		url:/users
		method:post
		data:action=login
		作用：根据该用户输入的信息，查询数据库中的user表，判断该用户输入的数据是否存在数据库中，返回查询结果

	3.向数据库中插入邮件
		url:/emails
		method:post
		data:action=write
		作用：向数据库中的emial表插入邮件，判断该用户输入的数据是否存在数据库中,返回是否插入成功
	4.查看邮件
		url:/emails
		method:post
		data:action=read
		作用：查询数据库中的emial表，返回该用户的邮件信息
